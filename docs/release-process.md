# Release process

## Versioning

The CLI follows [Semantic Versioning](https://semver.org):

- **MAJOR** — breaking CLI surface changes (renamed commands, removed
  flags, changed exit codes).
- **MINOR** — new commands, new flags, new JSON fields (additive).
- **PATCH** — bug fixes, doc updates, cassette refreshes.

The JSON envelope `schema` identifier (`robinhood-cli/v1`) only bumps on
breaking schema changes — e.g. renaming a field, removing a field,
changing a field's type. Adding optional fields is backwards-compatible
and does **not** bump the schema.

## Prerequisites (first release only)

Before the first release, set `HOMEBREW_TAP_TOKEN` in the repo secrets
to a fine-grained PAT scoped to `herocod3r/homebrew-tap` with
Contents: Read+Write.

## Steps

1. Run the full local gate:

   ```bash
   make test && make test-race && make test-fixtures \
     && make test-contract && make lint
   ```

2. Run `make release-dry-run` (wraps `goreleaser release --snapshot
   --clean --skip=publish`). Verify archives include `LICENSE`,
   `README.md`, `CHANGELOG.md`, and the `skills/{claude-code,codex}/`
   trees.

3. Run through [`docs/E2E.md`](./E2E.md) on macOS **and** Linux. Paste
   the completed checklist into the release PR body.

4. Bump `CHANGELOG.md`:
   - Rename the `[Unreleased]` heading to `[X.Y.Z] - YYYY-MM-DD`.
   - Add a new empty `[Unreleased]` section above it.

5. Commit and push a release PR. Await review.

6. Tag and push:

   ```bash
   git tag -a vX.Y.Z -m "vX.Y.Z"
   git push origin vX.Y.Z
   ```

7. GitHub Actions `release.yaml` runs GoReleaser on the tag. Verify
   archives for 6 platforms (`darwin` / `linux` / `windows` ×
   `amd64` / `arm64`) plus `checksums.txt` land on the GitHub release
   page.

8. Verify the Homebrew tap PR opened on `herocod3r/homebrew-tap` and
   merge it.

9. Test `brew install herocod3r/tap/rh` on a clean macOS VM:

   ```bash
   brew install herocod3r/tap/rh
   rh version --json | jq .data.version   # "X.Y.Z"
   ```

10. Re-snapshot the schema baseline so nightly drift checks run against
    the shipped version:

    ```bash
    go run ./cmd/rh commands --json > .github/schema-baseline.json
    git add .github/schema-baseline.json
    git commit -m "chore(ci): refresh schema baseline for vX.Y.Z"
    git push
    ```

11. Announce — write GitHub release notes (auto-generated by
    GoReleaser, manually polished), post in relevant channels.

## Hotfix process

1. Branch `hotfix/vX.Y.Z` off the previous tag.
2. Cherry-pick or write the fix; include regression test.
3. Run `make test && make lint && make release-dry-run`.
4. Skip the full E2E — re-run the commands that exercise the fix plus
   a portfolio smoke test.
5. Tag, push, follow steps 6-10 above.
